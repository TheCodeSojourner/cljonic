Detailed Description

template<ValidCljonicContainerElementType T, SizeType MaxElements>
class cljonic::Array< T, MaxElements >

The Array type is a fundamental immutable collection type in cljonic. It is implemented as a C array, and does not use heap memory. An Array has a specified maximum number of ordered elements each of the same specified type (i.e., It is homogenous). Note that one could create an Array of a UNION to get something like heterogeneity. An Array is a function of its indexable elements. An Array called with an out-of-bounds index will return its default element. Many Core functions accept Array arguments.

The Array constructor returns an instance of Array initialized with its arguments.

#include "cljonic.hpp"
using namespace cljonic;
int main()
{
constexpr auto a0{Array<int, 10>{}}; // immutable and empty
constexpr auto a1{Array<int, 10>{1, 2, 3, 4}}; // immutable and sparse
constexpr auto a2{Array<int, 4>{1, 2, 3, 4}}; // immutable and full
constexpr auto a4{Array{1, 2, 3, 4}}; // immutable and full of four int values
// Compiler Error: Array initialized with too many elements
// constexpr auto a{Array<int, 4>{0, 2, 4, 5, 6, 7, 8, 9}};
// Compiler Error: Attempt to create an Array bigger than CLJONIC_COLLECTION_MAXIMUM_ELEMENT_COUNT
// constexpr auto a{Array<int, 1111>{0, 2, 4, 5, 6, 7, 8, 9}};
return 0;
}
