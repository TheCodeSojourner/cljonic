Detailed Description

template<typename T, std::size_t MaxElements>
class cljonic::Set< T, MaxElements >

The Set type is a fundamental immutable collection type in cljonic. It is implemented as a C array, and does not use dynamic memory. A Set has a specified maximum number of unique unordered elements each of the same specified type (i.e., It is homogenous). Note that one could create a Set of a UNION or std::variant to get something like heterogeneity. A Set is a function of its elements. A Set called with a value not contained within the set will return its default element. Many Core functions accept Set arguments.

template<typename T , std::size_t MaxElements>
using cljonic::Set< T, MaxElements >::cljonic_collection_type = std::integral_constant<CljonicCollectionType, CljonicCollectionType::Set>

The Set constructor returns an instance of Set initialized with the unique elements in its arguments. If the number of unique elements in its arguments exceeds the maximum number of elements, the extra unique elements in its arguments are silently ignored.

#include "cljonic.hpp"
using namespace cljonic;
int main()
{
const auto s0{Set<int, 10>{}}; // immutable, empty
const auto s1{Set<int, 10>{1, 2, 3, 4}}; // immutable, sparse
const auto s2{Set<int, 4>{1, 2, 3, 4}}; // immutable, full
const auto s3{Set<int, 4>{1, 2, 3, 4, 5, 6}}; // immutable, full, but 5 and 6 are ignored
const auto s4{Set<int, 4>{1, 2, 1, 4, 5, 6}}; // immutable, full with 1, 2, 4, 5
const auto s5{Set{1, 2, 3, 4}}; // immutable, full of four int values
const auto s6{Set{1, 2, 1, 4}}; // immutable, full of three unique int values
return 0;
}
